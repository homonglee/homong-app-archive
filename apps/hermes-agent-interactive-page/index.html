One core, many surfaces

하나의 에이전트를 여러 접점에서 사용합니다.

CLI에서 시작한 일을 Telegram에서 이어가고, IDE에서 코드 작업을 맡기고, Cron으로 자동화할 수 있습니다.

Tools, not just text

Hermes는 실제 도구를 호출합니다.

파일, 터미널, 웹, 브라우저, GitHub, 이메일, 문서, 이미지 생성, 자동화까지 워크플로우 안의 도구들을 직접 사용합니다.

Hermes Agent Coretool calling · routing · sessions

Self-improving layer

매번 처음 만나는 에이전트가 아닙니다.

Memory, Skills, Sessions가 사용자의 선호와 환경, 반복 절차, 과거 대화를 축적합니다.

Provider agnostic

모델은 바꿔도 워크플로우는 유지됩니다.

OpenAI, Anthropic, OpenRouter, Google, xAI, 로컬 모델 등 다양한 provider를 연결할 수 있습니다.

Automation that comes back

예약하고, 실행하고, 원하는 채널로 받습니다.

정기 리포트, 블로그 모니터링, CI 알림, 뉴스 요약 같은 일을 Cron과 Gateway로 자동화합니다.

Telegram preview

Extensible by design

필요한 도구를 붙여 나만의 에이전트를 만듭니다.

플러그인, MCP 서버, Custom Tool, Profiles, Python 생태계를 통해 Hermes를 원하는 방식으로 확장할 수 있습니다.
